 웹앱(Web App) 개발 계획 초안
사용자가 카테고리별로 앱을 확인하고 즉시 다운로드 페이지로 이동할 수 있는 "Korea Starter Pack (가칭)" 웹앱 개발 계획입니다.
1단계: 기획 및 요구사항 정의 (Planning)
• 타겟 사용자: 한국을 방문하는 여행객 및 장기 거주 외국인 (유학생, 근로자 등).
• 핵심 가치: "한국 생활의 장벽(언어, 정보, 교통)을 앱 하나로 해결하는 게이트웨이".
• 주요 기능:
    ◦ 카테고리별 앱 리스트 뷰 (아이콘, 이름, 간략 설명).
    ◦ 원클릭 다운로드 링크: 각 앱 카드 클릭 시 사용자의 OS(iOS/Android)를 감지하여 앱스토어/플레이스토어로 자동 리다이렉트.
    ◦ 다국어 지원: 영어(기본), 중국어, 일본어 등 UI 언어 변경 기능.
2단계: 기술 스택 선정 (Tech Stack)
개발 속도와 유지보수 편의성을 고려한 스택입니다.
• Frontend: React 또는 Vue.js (컴포넌트 기반으로 리스트 관리 용이), Tailwind CSS (빠른 UI 디자인).
• Deployment: Vercel 또는 Netlify (무료 호스팅 및 빠른 배포).
• Data Management: 간단한 JSON 파일로 앱 데이터(이름, 링크, 설명, 이미지 경로) 관리. 백엔드 서버 없이 정적 사이트 생성(SSG) 방식 추천.
• PWA (Progressive Web App): 웹사이트지만 앱처럼 홈 화면에 설치하여 사용할 수 있도록 PWA 기능 적용.
3단계: UI/UX 디자인 전략 (Design)
• 직관적인 아이콘 사용: 텍스트를 읽지 않아도 '택시', '배달', '지도' 등을 알 수 있는 픽토그램 활용.
• 정보의 단순화: 앱 설명은 "Best for Taxi" 또는 "Subway Info"처럼 핵심 키워드 위주로 짧게 구성.
• 신뢰도 강화: "정부 추천(Emergency Ready)" 또는 "현지인 필수(KakaoTalk)" 뱃지 부착.
4단계: 데이터 수집 및 링크 구축 (Data Construction)
• Deep Link 확보: 각 앱의 구글 플레이스토어 및 애플 앱스토어 URL 수집.
• OS 감지 로직: 웹앱 접속 기기가 안드로이드면 플레이스토어, 아이폰이면 앱스토어로 연결되도록 자바스크립트 로직 구현. (예: navigator.userAgent 활용)
5단계: 확장 및 홍보 (Future Scope)
• 커뮤니티 연동: 추후 '외국인 커뮤니티' 링크 추가 (소스의 '스테이포틴' 등 참고).
• 팁 섹션 추가: "택시 타는 법", "배달 시 주소 입력 팁" 등 앱 사용 가이드 콘텐츠 추가.
이 계획안은 초기 개발 비용을 최소화하면서도 외국인 사용자에게 즉각적인 가치를 제공할 수 있는 구조로 작성되었습니다.
